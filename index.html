FDMS API

Understanding ZIMRA's Fiscal Data Management System

FDMS API Overview

Real-time Fiscal Integration

The FDMS API enables direct communication between fiscal devices and ZIMRA's central system. This allows for real-time transaction monitoring and reporting to improve tax compliance.

REST API JSON Format SSL Encrypted

API Authentication

Secure Access Protocols

All API interactions require secure authentication. Businesses must register their devices to receive API credentials, including unique device IDs and digital signatures for transaction verification.

OAuth 2.0 JWT Tokens Key Rotation

Transaction Endpoints

Core API Functions

The API provides endpoints for submitting sales transactions, issuing fiscal receipts, generating Z-reports, and synchronizing tax rates. All transactions receive a fiscal signature from ZIMRA for validation.

/transactions /receipts /reports
Virtual Solutions

Virtual Devices System

ZIMRA's digital fiscal management solution

Virtual Fiscal Devices

Digital Solution

ZIMRA's Virtual Devices System allows businesses to use software-based fiscal solutions instead of physical devices. This cloud-based system offers flexibility while ensuring tax compliance for all transactions.

Electronic Receipting

Real-time Compliance

The system generates fiscalized electronic receipts with QR codes that can be verified through ZIMRA's portal. Each receipt contains a unique fiscal signature validating the transaction with tax authorities.

Reporting Dashboard

Business Intelligence

Businesses gain access to a comprehensive reporting dashboard showing sales trends, tax obligations, and compliance status. The system automatically generates VAT returns and fiscal reports required by ZIMRA.

Integration Options

Flexible Implementation

The system offers multiple integration options including direct API access, SDK libraries for popular programming languages, and ready-made plugins for common POS and ERP systems used in Zimbabwe.
